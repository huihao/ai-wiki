# AI-Friendly Codebase Design

## Summary

AI-friendly codebase design refers to the practice of structuring and organizing software repositories so that AI coding agents and assistants can more effectively understand, navigate, and modify the code. This includes conventions for file organization, naming, documentation, context files (like AGENTS.md or CLAUDE.md), and modular architecture that reduces ambiguity for AI tools.

## Key Attributes
- **Type**: software engineering practice / design philosophy
- **Notable for**: enabling AI coding agents to work more autonomously and accurately by reducing context-gathering overhead
- **Other facts**: key practices include explicit project documentation, clear module boundaries, consistent naming conventions, and context files that describe project structure; reduces the number of steps AI agents need to take before making edits

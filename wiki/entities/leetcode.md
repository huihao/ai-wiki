# LeetCode

Online platform for coding interview practice, featuring algorithm and data structure problems ranging from easy to hard difficulty.

## Purpose

Prepare for technical interviews at software companies by:
- Building muscle memory for common patterns
- Recognizing algorithmic techniques quickly
- Practicing coding under time constraints
- Developing communication skills for "talk out loud" interviews

## Problem Difficulty

- **Easy**: Warm-up problems, basic patterns
- **Medium**: Most common interview difficulty, multi-step solutions
- **Hard**: Complex algorithms, optimization challenges

## Preparation Strategy

Based on [[max-mynter-research-engineer-career|Max Mynter's approach]]:

### Foundation Building (150-200 problems)
- Build repertoire of algorithmic techniques
- Learn useful data structures and their operations
- Develop pattern recognition across problem types

### Rapid Practice (20-30 problems before interviews)
- Quick warm-up for muscle memory
- Focus on common patterns likely to appear
- Time-boxed practice for interview conditions

### Resources
- **NeetCode 150**: Ordered roadmap introducing concepts progressively
- **Skiena's Algorithm Design Manual**: Theoretical foundation
- **Coding Interview Patterns**: Practical interview-focused approach

## Problem Types

Common categories appearing in interviews:
- Array manipulation
- String processing
- Tree/graph traversal
- Dynamic programming
- Two-pointer techniques
- Sliding window patterns
- Binary search variants

## Interview Formats

- **Online interpreters**: Coderpad, Google Docs
- **Screen sharing**: Own editor or platform interpreter
- **Language choice**: Python recommended for algorithmic interviews (avoids memory management)

## Performance Metrics

Based on text classification experiment comparison:
- Custom implementations: slower but correct (218s vs 155s PyTorch)
- Accuracy parity validates correctness despite speed differences

# Ryo Kusnadi

## Summary

Author of a practical guide to implementing Harness Engineering using a minimal Go project (crypto snapshot API) as a running example. Introduced a 5-level maturity model (L0-L4) and broke down the core components: Rules (soft constraints), Skills (SOPs), Sub-Agents (role division), and MCP (tool integration).

## Key Attributes

- Published Harness Engineering guide on GitConnected (2026-05-11)
- Practical approach focused on "what to build first and in what order"

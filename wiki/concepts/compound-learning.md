# Compound Learning

Progressive accumulation of knowledge and patterns that makes each subsequent improvement easier than previous ones.

## Core Principle

"Each improvement should make future improvements easier" - Compound Product philosophy

## Mechanisms

### Memory Persistence

- **Git History**: Code changes provide context
- **Progress Logs**: Chronological record of attempts
- **Task State**: Status tracking prevents rework
- **AGENTS.md**: Long-term semantic memory

### Knowledge Accumulation

- Document discovered patterns
- Record gotchas and mistakes
- Store project conventions
- Archive style preferences

## Implementation

### AGENTS.md Sections

1. Patterns & Conventions
2. Gotchas (past mistakes)
3. Style/Preferences
4. Recent Learnings

### Feedback Loops

- Agent records mistakes in AGENTS.md
- Future iterations read and avoid repeating
- Human corrections added as persistent rules

## Benefits

- Agent effectiveness increases over time
- Fewer repeated mistakes
- Faster convergence on correct patterns
- Knowledge transfer across agent runs

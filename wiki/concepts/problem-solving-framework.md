# Problem Solving Framework

## Definition

A structured approach to solving programming problems, breaking down the process into four phases: understanding, planning, implementing, and reflecting. The framework emphasizes deliberate problem analysis before coding, systematic implementation, and post-solution optimization. It contrasts with jumping directly into code without understanding the problem.

## Key Attributes

- **Phase 1: Understand the problem** — identify inputs, outputs, constraints
- **Phase 2: Plan** — pseudocode, algorithm design, breakdown
- **Phase 3: Implement** — code incrementally, test frequently
- **Phase 4: Reflect** — optimize, refactor, generalize, learn
- **Cost per phase**: Understanding/planning cheap; fixing misunderstood problems expensive

## Process Details

### Understanding Phase
- Read problem statement carefully
- Identify what's given (inputs)
- Identify what's required (outputs)
- Clarify constraints and edge cases
- "Think like a villain" — anticipate failure modes

### Planning Phase
- Write pseudocode
- Break into subproblems
- Consider proof of concept for risky parts
- Estimate complexity and feasibility

### Implementation Phase
- Code incrementally
- Test each piece before moving on
- Keep backup of working versions

### Reflection Phase
- Can it be optimized?
- Can it be generalized?
- What did you learn?
- How could approach be improved next time?

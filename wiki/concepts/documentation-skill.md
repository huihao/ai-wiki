# Documentation Skill

## Definition

Documentation skill refers to the ability to produce high-quality, well-structured technical documentation by applying systematic frameworks for organizing content by purpose and audience. The Diataxis framework is the canonical approach, classifying documentation into four types: tutorials (learning-oriented), how-to guides (task-oriented), explanation (understanding-oriented), and reference (information-oriented).

## Key Aspects

- The Diataxis framework provides four quadrants defined by two axes: practitioner vs. learner orientation, and knowledge vs. action orientation
- Tutorials are learning-oriented: guided lessons that teach a concept step by step, suitable for newcomers
- How-to guides are task-oriented: practical steps to accomplish a specific goal, assuming the reader has basic knowledge
- Explanation content is understanding-oriented: provides background, context, and rationale behind design decisions
- Reference documentation is information-oriented: precise, complete, and accurate descriptions of APIs, commands, and data structures
- Good documentation skills recognize that each type serves a different purpose and should not be conflated
- Applying the framework prevents common documentation failures: reference disguised as tutorials, or how-to guides missing enough context

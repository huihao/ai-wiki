# Vim Keybindings

## Definition

Vim keybindings are keyboard shortcuts and interaction patterns inspired by the Vim text editor, characterized by modal editing and efficient keyboard-only navigation. Vim operates in distinct modes -- Normal (for navigation and commands), Insert (for typing text), Visual (for selecting text), and Command-line (for Ex commands) -- which allow users to compose powerful editing operations without reaching for a mouse. The keybinding philosophy prioritizes minimizing hand movement and maximizing editing speed through composable verb-noun commands (e.g., `d` for delete, `w` for word, so `dw` deletes a word). Beyond text editors, Vim keybindings have been adopted in browser extensions, terminal multiplexers (tmux), file managers, and AI coding tools, forming a cross-tool productivity ecosystem.

## Evolution

- **1991**: Vim 1.0 is released by Bram Moolenaar, inheriting Vi's modal editing model from 1976
- **2000s**: Vim plugins (NERDTree, vim-fugitive) extend keybinding patterns to IDE-like workflows
- **2010s**: Browser extensions (Vimium, Surfingkeys) bring Vim keybindings to web browsing
- **2015**: VS Code Vim extension makes Vim keybindings available in modern editors
- **2020s**: AI coding tools and terminal applications adopt Vim keybindings as a power-user interface
- **2024+**: Vim keybindings become a standard input mode in AI coding assistants (Claude Code, Cursor)

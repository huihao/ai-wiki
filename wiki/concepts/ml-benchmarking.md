# ML Benchmarking

## Definition

ML benchmarking is the practice of evaluating and comparing machine learning models using standardized datasets, metrics, and evaluation protocols to measure performance, reproducibility, and progress across the field.

## Key Aspects

- The UC Irvine Machine Learning Repository has served the community since 1987 with 689 curated datasets spanning biology, medicine, finance, and social sciences
- Classic benchmarks include Iris (1936), Heart Disease, Wine Quality, Breast Cancer Wisconsin, and Adult/Census Income datasets
- Standardized metrics (PSNR, SSIM, mAP, F1, FID) enable fair cross-study comparisons, but inconsistent evaluation protocols remain a persistent problem
- New datasets are continuously added (Amazon/Google reviews, medical recordings, vision benchmarks) to address emerging task types
- Benchmarking pitfalls include using weak baselines, ignoring variance, and overfitting to specific benchmarks
- Repositories like UCI and Kaggle provide both data and leaderboards for systematic comparison
- Reproducibility requires documenting task/data/metrics/scripts in structured cards

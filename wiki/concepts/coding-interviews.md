# Coding Interviews

## Definition

Coding interviews are technical assessment sessions where candidates solve algorithmic and data structure problems under time pressure, typically using an online platform like LeetCode. They test pattern recognition, problem decomposition, coding fluency, and the ability to articulate thinking while writing correct solutions.

## Key Aspects

- **Platform**: LeetCode provides structured problems from Easy to Hard with progressive difficulty
- **Preparation Strategy**: 150-200 foundational problems, then 20-30 rapid warm-ups before interviews
- **Problem Categories**: Arrays, strings, trees/graphs, dynamic programming, two-pointer, sliding window, binary search
- **Pattern Recognition**: Successful candidates recognize algorithmic patterns rather than solving each problem from scratch
- **Resources**: NeetCode 150 (ordered roadmap), Skiena's Algorithm Design Manual, Coding Interview Patterns
- **Interview Formats**: Coderpad, Google Docs, screen sharing with language choice (Python recommended for algorithmic focus)
- **Communication**: Talking through approach and reasoning is as important as the code itself
- **Time Management**: Solving medium-difficulty problems in 20-30 minutes is the benchmark

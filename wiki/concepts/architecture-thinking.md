# Architecture Thinking

## Definition

Architecture thinking is the discipline of system design and structural planning as the core skill for AI-assisted coding, going beyond prompt engineering to focus on how components connect, how complexity is decomposed, and how systems are planned before execution begins. It represents the most critical skill for developers working with AI coding agents.

## Key Aspects

- Architecture over prompts: the real skill in AI-assisted coding is system design, not prompt crafting
- Breaking down complexity into manageable, well-defined components enables agents to work effectively
- Understanding how components connect prevents integration failures that AI agents often introduce
- Planning before executing reduces costly rework — AI is fast at implementation but poor at architectural decisions
- System design thinking includes considering data flow, error boundaries, and extensibility from the start
- Good architecture makes AI coding agents more effective by providing clear structural constraints
- Contrasted with prompt engineering: prompts are tactical, architecture is strategic

# Harness Engineering Is Cybernetics

## Definition

Harness Engineering Is Cybernetics is the thesis that harness engineering — designing the environment, constraints, and validation systems that make AI agents reliable — is fundamentally an implementation of cybernetics, the science of control and communication in systems. The framework maps Norbert Wiener's cybernetic principles (information, control, feedback) directly onto AI coding practices, treating the AI agent as a system to be controlled rather than a tool to be directed.

## Key Aspects

- **Control Loop Mapping**: AI coding follows the cybernetic loop: [Set goal] -> [Sense deviation] -> [Apply intervention] -> [Eliminate deviation]
- **Information Reduces Uncertainty**: In cybernetics, information acquisition reduces entropy; in harness engineering, structured context (Specs, AGENTS.md, tool results) reduces the agent's uncertainty
- **Negative Feedback for Self-Correction**: The core mechanism for reliable AI systems is negative feedback — detecting when the agent deviates from desired behavior and correcting it
- **Sensors and Actuators**: MCP tools serve as the agent's sensors (reading state) and actuators (modifying state); lint rules, tests, and type checkers are business-agnostic sensors
- **Domain Modeling as Communication**: Structured requirements (Specs) communicate intent to AI more precisely than natural language; domain modeling becomes more important, not less
- **Self-Reproducing Systems Risk**: Bad code generated by AI can be referenced and copied by future AI sessions, creating compounding errors analogous to nuclear chain reactions

# Git Workflows

## Definition

Git workflows are structured processes and conventions for using Git version control in collaborative software development. They define branching strategies, merge policies, code review practices, and release management procedures that teams follow to maintain code quality and coordination across contributors.

## Key Aspects

- **Branching Strategies**: Different workflows define how feature branches, release branches, and hotfix branches are created, merged, and deleted (e.g., Git Flow, GitHub Flow, Trunk-Based Development)
- **Pull Request Model**: Most modern workflows center on pull requests (or merge requests) as the primary mechanism for code review and integration
- **CI/CD Integration**: Workflows increasingly embed automated testing, linting, and deployment checks that run on every branch push or pull request
- **AI Agent Integration**: Modern coding agents operate within Git workflows, creating branches, making commits, and submitting pull requests autonomously
- **CLI Tools**: GitHub CLI (`gh`) streamlines workflow operations — creating PRs, viewing checks, managing issues — from the command line, making it well-suited for agent automation

# Checklists

## Definition
Checklists in the context of spec-driven AI development are structured, step-by-step tracking mechanisms used to ensure completeness of requirements, design decisions, and implementation tasks. They serve as lightweight governance tools that guide both humans and AI coding agents through complex multi-step workflows.

## Key Aspects
- GitHub's spec-kit makes heavy use of checklists for tracking progress through specification and implementation phases
- Checklists bridge the gap between high-level specs and executable tasks, providing verifiable completion criteria
- In SDD (Spec-Driven Development), checklists track the progression: Constitution -> Specify -> Plan -> Tasks
- Each task in a spec can have associated checklists that define acceptance criteria (GIVEN/WHEN/THEN format)
- Checklists serve as memory anchors for AI agents, providing structured context that persists across sessions
- Different from memory banks (general project context) — checklists are specific to particular features or stories

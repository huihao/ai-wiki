# SOAR Architecture

## Definition

A cognitive architecture proposed in 1987 by Allen Newell, Paul Rosenbloom, and John Laird that unified problem-solving, learning, and memory into a single framework. SOAR distinguished between working memory (declarative knowledge for immediate tasks) and long-term procedural memory (production rules for learned strategies), and introduced chunking — the automatic creation of new production rules from successful problem-solving episodes.

## Key Features

- **Declarative knowledge** (facts/info) in working memory
- **Procedural knowledge** (action rules) in long-term memory
- **Chunking**: automatic compilation of experiences into reusable rules
- **Subgoaling**: hierarchical decomposition of complex problems
- General intelligence as handling a full range of cognitive tasks

## Historical Significance

- Pioneered the separation of immediate task storage from learned rules
- Anticipated modern agentic memory challenges
- Conceptual ancestor to today's agent frameworks

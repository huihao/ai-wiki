# Machine Learning Datasets

## Definition

Machine learning datasets are curated collections of data used for training, testing, and benchmarking machine learning models. Datasets typically include labeled examples for supervised learning, unlabeled data for unsupervised learning, or structured inputs for specific tasks.

## Key Repositories

- [[uc-irvine|UC Irvine ML Repository]] — 689 classic datasets (Iris, Heart Disease, Wine)
- Kaggle — Modern platform with competitions and datasets
- Hugging Face Datasets — NLP-focused datasets
- ImageNet — Large-scale image dataset
- OpenML — Collaborative ML dataset platform

## Dataset Types

- **Classification**: Predict categorical labels (e.g., Iris, Breast Cancer)
- **Regression**: Predict continuous values (e.g., Wine Quality)
- **Clustering**: Group similar instances (unlabeled)
- **Time Series**: Sequential/temporal data (e.g., Bank Marketing)
- **Vision**: Image/video data (e.g., COREVQA)
- **Text/NLP**: Language data for text tasks
- **Medical**: Healthcare applications (e.g., Heart Disease, HLS-CMDS)

## Classic Examples

- Iris (1936): 150 flower samples, 3 classes
- MNIST: Handwritten digit images
- Adult/Census Income: Demographic data for income prediction
- Wine Quality: Chemical properties and quality ratings

## Evolution

Dataset repositories evolved from early academic collections (UCI, 1987) to modern platforms (Kaggle, HuggingFace). Early datasets like Iris became ML benchmarks. Modern trends: larger scale, multi-modal data, domain-specific collections. Dataset quality and representativeness crucial for model performance.

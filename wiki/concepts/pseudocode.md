# pseudocode

## Definition

Pseudocode is an informal, human-readable description of an algorithm that uses the structural conventions of programming (loops, conditionals, function calls) without adhering to any specific programming language's syntax. It serves as an intermediate planning step between understanding a problem and writing working code, enabling developers to work through logic and identify edge cases before committing to implementation details. Pseudocode is central to systematic problem-solving frameworks taught in computer science education.

## Key Aspects

- **Planning Before Coding**: Listed as the second step in the problem-solving framework -- after understanding the problem, you write pseudocode before implementing in any language
- **Language Agnostic**: Expresses logic without committing to syntax details, making it useful for communicating algorithms across language boundaries
- **Reduces Cognitive Load**: Allows focus on algorithmic logic rather than language-specific concerns (error handling, imports, types)
- **Aids Debugging**: Bugs in pseudocode are cheaper to fix than bugs in working code; reviewing pseudocode catches logical errors early
- **Communication Tool**: Serves as a shared language between developers who may work in different languages
- **Teaching Aid**: Introduced early in CS education (e.g., Beej's Guide to Learning Computer Science) to build algorithmic thinking before language fluency
- **Edge Case Identification**: The "think like a villain" step in problem-solving explicitly anticipates edge cases during pseudocode review

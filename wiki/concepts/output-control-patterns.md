# Output Control Patterns

## Definition

Output control patterns are systematic approaches to constraining and directing the text generated by large language models, operating at different stages of the generation pipeline to enforce brand consistency, format compliance, and content safety without relying on post-generation retry loops.

## Key Aspects

- **Perception-Decision-Execution Framework**: Taobao's approach abstracts human operations into an AI closed loop where output control is enforced at the decision stage through workflow or agent modes
- **Three Automation Levels**: Tool Assistant (simple single-task analysis), Complex Workflow (multi-step serial processes with fixed flows and conditional branches), and Intelligent Agent (autonomous planning with dynamic tool selection)
- **Workflow vs. Agent Decision Modes**: Workflow mode uses predefined step sequences for predictable tasks; Agent mode uses adaptive planning for tasks requiring flexible investigation paths
- **Accuracy Assurance**: Prompt engineering with quantifiable criteria, conservative strategies for high-risk operations (mark "needs human confirmation" when confidence is low), and structured JSON output for traceability
- **Human-in-the-Loop**: Inserting observation nodes at critical steps for manual verification, particularly for high-risk operations where AI confidence is insufficient
- **Graduated Complexity**: Start with simplest automation (Tool Assistant), evolve to Complex Workflow, and only adopt Intelligent Agent patterns when task variability demands it

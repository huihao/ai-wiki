# Technical Writing

## Definition

Technical writing is the practice of creating clear, accurate, and structured documentation that communicates complex technical information to specific audiences. In the context of software and AI systems, technical writing encompasses API documentation, tutorials, how-to guides, reference manuals, and architectural decision records, following frameworks like Diataxis to ensure documentation serves distinct user needs systematically.

## Key Aspects

- **Diataxis framework**: A systematic approach to documentation that identifies four fundamental types — tutorials (learning-oriented), how-to guides (task-oriented), explanation (understanding-oriented), and reference (information-oriented) — each serving a different user need and requiring different writing approaches.
- **Audience awareness**: Effective technical writing adapts tone, depth, and terminology to the target reader — a getting-started tutorial for beginners differs fundamentally from an API reference for experienced developers.
- **Task-oriented writing**: How-to guides and tutorials are organized around what the reader wants to accomplish, not around the system's internal structure, reducing cognitive load and enabling practical application.
- **Composability of documentation**: Good technical documentation is modular — individual pages can be read independently and composed into larger learning paths, mirroring how users actually consume documentation.
- **Documentation-as-code**: Modern technical writing treats documentation as version-controlled artifacts alongside source code, enabling collaborative editing, review, and automated deployment.
- **Quality markers**: Clear structure (headings, lists, code examples), concise language, accurate examples, and maintenance discipline distinguish effective technical writing from mere information dumping.

## Summary

Exactly one year after Karpathy coined "vibe coding," he announced its successor: "agentic engineering." This article defines the distinction: vibe coding is YOLO (no review), while agentic engineering means AI builds under human ownership of architecture, quality, and correctness.

## Key Takeaways

- Karpathy's Feb 4 2026 tweet: "agentic engineering" is his preferred term for professional LLM agent programming.
- Vibe coding = going with vibes, not reviewing code. Agentic engineering = AI builds, human owns.
- Three failure modes of vibe coding at scale: security holes, unmaintainable architecture, context collapse.
- Agentic engineering workflow: write spec first → break into scoped tasks → review like a human PR → test relentlessly.
- Results from companies doing it right: TELUS (500K+ hours saved), Zapier (89% adoption), Stripe (1,000+ merged PRs/week).
- New developer role: architect, not author. Define what gets built, design architecture, review agent output, decide what ships.

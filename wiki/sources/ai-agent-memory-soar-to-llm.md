# AI Agent Memory Explained: From SOAR to LLM Memory Mode

## Summary

Traces the evolution of AI agent memory from the 1987 SOAR cognitive architecture through modern LLM memory systems. Explains how SOAR's distinction between working memory and long-term procedural memory anticipated modern agentic challenges, and covers current memory types (semantic, episodic, procedural), generative agent memory implementations, and how ChatGPT's memory mode works.

## Key Takeaways

- SOAR (1987, Newell/Rosenbloom/Laird) pioneered the separation of working memory (declarative knowledge) from long-term procedural memory (production rules)
- SOAR's chunking mechanism automatically creates new production rules from successful problem-solving episodes — analogous to modern experience-based learning
- Modern agentic AI relies on statistical learning and vector-based retrieval rather than explicit production rules, but the fundamental question of how systems remember remains the same
- Memory is the dynamic record of experience that threads together profiling (who the agent is) and knowledge (what the agent knows)
- Current AI systems can retrieve, summarize, and store selective details but lack stable, structured memory that persists reliably over time
- ChatGPT's memory mode stores selective facts across conversations but is not a true persistent memory system

## Key Concepts

- **Declarative knowledge** (facts/info) held in working memory vs **procedural knowledge** (action rules) in long-term memory
- **Chunking**: automatic compilation of problem-solving episodes into reusable rules
- **Subgoaling**: hierarchical decomposition of complex problems into manageable subtasks
- Three memory types in modern agents: semantic, episodic, procedural

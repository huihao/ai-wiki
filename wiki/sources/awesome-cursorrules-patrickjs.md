# Awesome Cursor Rules

> Configuration files that enhance Cursor AI editor experience with custom rules and behaviors.

**Source:** [GitHub - PatrickJS/awesome-cursorrules](https://github.com/PatrickJS/awesome-cursorrules) by PatrickJS (September 16, 2024)

---

## Overview

Collection of `.cursorrules` configuration files for customizing [[Cursor]] AI code editor behavior. `.cursorrules` files define project-specific instructions for AI to follow when generating code.

---

## Why Use .cursorrules?

### Benefits

- **Customized AI Behavior** — tailor responses to project-specific needs
- **Consistency** — enforce coding standards and best practices
- **Context Awareness** — provide project context (methods, architecture, libraries)
- **Improved Productivity** — generate code requiring less manual editing
- **Team Alignment** — shared rules ensure consistent AI assistance across team
- **Project-Specific Knowledge** — include structure, dependencies, unique requirements

## Installation

### Manual Method

1. Install [[Cursor]] AI
2. Browse rules in repository
3. Copy `.cursorrules` to project root directory
4. Customize rules as needed

### VSCode Extension Method

1. Install [[Cursor]] AI
2. Install [vscode-cursor-rules](https://marketplace.visualstudio.com/items?itemName=BeilunYang.cursor-rules) extension
3. Open command palette (Cmd+Shift+P / Ctrl+Shift+P)
4. Type `Cursor Rules: Add .cursorrules`
5. Select and download desired `.cursorrules` file
6. Customize rules

## Contribution Guidelines

**Folder naming pattern:**
```
technology-focus-cursorrules-prompt-file
```

**Example:**
```
react-typescript-cursorrules-prompt-file
```

**Steps:**
1. Fork repository
2. Create new folder in `rules/` directory
3. Add `.cursorrules` file
4. Optionally include README.md for credit/description
5. Update main README.md with contribution
6. Follow `.cursorrules` guidelines at repository root
7. Submit pull request

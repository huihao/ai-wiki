# How to Get From High School Math to Cutting-Edge ML/AI

**Author:** Justin (justinmath.com)
**URL:** https://www.justinmath.com/how-to-get-from-high-school-math-to-cutting-edge-ml-ai/
**Published:** August 2024 (X/Twitter traction)

## Summary

A detailed 4-stage roadmap for software engineers to go from high school math to reading cutting-edge ML papers. The author emphasizes that foundational math is the "make-or-break moment," and that classical ML knowledge provides intuition patterns that make modern papers readable. Each stage includes specific resource recommendations.

## Key Takeaways

1. **Four stages**: (1) Foundational Math → (2) Classical ML → (3) Deep Learning → (4) Cutting-Edge ML
2. **Stage 1 — Foundational Math**: Algebra, single-variable calculus, linear algebra, probability/statistics, multivariable calculus (gradients + chain rule only, skip divergence/curl/Stokes). ~70 hours via Math Academy.
3. **Stage 2 — Classical ML**: Code streamlined versions of models from linear regression to small neural networks. Build pattern recognition for loss functions, gradient descent, backprop. Resources: justinmath.com textbook + Andrew Ng's ML Specialization on Coursera.
4. **Stage 3 — Deep Learning**: Simon Prince's *Understanding Deep Learning* (free, highly rated) + Jeremy Howard's *Practical Deep Learning for Coders* (Fast.AI).
5. **Stage 4 — Cutting-Edge ML**: Read from basics up, not top-down. Andrej Karpathy's *Neural Networks: Zero to Hero* + Jeremy Howard's *From Deep Learning Foundations to Stable Diffusion*.
6. **Classical ML gives intuition for papers**: Loss functions, gradient descent patterns, model architecture patterns make equations in modern papers immediately recognizable.
7. **Deliberate practice > curiosity alone**: Curiosity motivates; deliberate practice builds ability. Play enough to stay motivated, spend most time building hard skills.
8. **No graduate-level math needed**: Undergraduate math suffices for implementing new models from papers. Advanced math only needed for theoretical ML research.
9. **SWE skills assumed**: The roadmap targets people already in software; CS skills can be picked up as needed since CS is "less hierarchical than math."
10. **Implement from scratch**: Key learning comes from coding streamlined versions of each model, not just watching lectures.

## Raw Notes

- The roadmap is specifically for SWE people wanting to read ML papers
- Math Academy covers all Stage 1 topics in ~70 hours if you know through single-variable calculus
- Free alternatives: Khan Academy (arithmetic through high school), OpenStax, MIT OCW
- The textbook "Introduction to Algorithms and Machine Learning" includes data structures/algorithms + classical ML
- Understanding Deep Learning by Prince: 4.8/5 on Amazon, blew up on HackerNews
- "The cutting edge is more of a zone where guidance gradually fades" — not a single line
- Bottom-up approach recommended over top-down for entering a new field

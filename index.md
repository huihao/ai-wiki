## Concepts

- [[agent-engineering|Agent Engineering]] — Agent Engineering is the systematic discipline of designing, building, and evaluating AI agents — systems where LLMs drive control flow, make decisions, and interact with tools or environments. It goes beyond simple prompt engineering to address planning, memory, authority delegation, and trust.
- [[agent-skills|Agent Skills]] — Reusable, declarative capability packages that give AI agents deterministic, professional-grade abilities. Skills serve as the bridge between an LLM's generalization and deterministic, reusable expertise — essentially microservices for agent capabilities.
- [[agent2agent-protocol|Agent2Agent Protocol (A2A)]] — An open JSON-RPC standard released by Google that enables AI agents to discover, authenticate, and exchange data across network boundaries. A2A complements the Model Context Protocol (MCP) by handling inter-agent communication rather than in-agent tool exposure.
- [[agentic-engine-optimization|Agentic Engine Optimization]] — The practice of structuring, formatting, and serving technical content so that AI coding agents can consume it effectively. AEO draws an analogy to SEO but targets autonomous agent consumers rather than human readers.
- [[agentic-engineering|Agentic Engineering]] — An engineering paradigm where humans act as coordinators rather than code authors, using systematic context management, knowledge沉淀, and process design to enable AI agents to work autonomously while maintaining quality. Coined by Andrej Karpathy in February 2026 as the disciplined successor to "vibe coding."
- [[agentic-search|Agentic Search]] — A context management strategy where agents maintain lightweight indexes and retrieve knowledge dynamically at runtime rather than pre-loading all knowledge into the context window. Coined by Anthropic in their context engineering guidance, it has been independently validated by OpenAI Codex, Cursor, and production teams.
- [[agentic-security|Agentic security]] — Agentic security refers to the security frameworks, boundaries, and risks associated with AI agents that can read files, execute code, and interact with systems autonomously. Core concerns include prompt injection, secret exfiltration, and sandboxing of generated code.
- [[agents-md|AGENTS.md]] — AGENTS.md is a context file used to provide persistent, project-specific knowledge to AI coding agents across iterations. It serves as a living notebook of conventions, gotchas, and non-discoverable information that agents need to work effectively in a codebase.
- [[ai-engineering|AI Engineering]] — The discipline of building production systems with large language models (LLMs) and other foundation models. Distinct from AI research (training new models) and traditional ML engineering (feature engineering, tabular models). AI Engineering focuses on prompting, RAG, agents, evals, finetuning, and orchestration of existing models into useful products.
- [[ai-first|AI-First]] — An organizational strategy that assumes AI (specifically autonomous agents) is the primary builder and executor, with human roles redesigned around direction, judgment, and exception handling. Contrast with "AI-assisted," where AI tools are added to existing human-centric workflows without structural change. The difference is multiplicative rather than incremental.
- [[ai-ide-customization|AI IDE Customization]] — The practice of tailoring AI-powered integrated development environments (IDEs) to align with project-specific coding standards, workflows, and toolchains. Customization ensures AI assistants generate code that matches team conventions and can interact with external systems.
- [[ai-in-software-testing|AI in Software Testing]] — The application of artificial intelligence — including machine learning, natural language processing, and computer vision — to automate, optimize, and enhance software quality assurance processes. AI in testing spans test case generation, execution, maintenance, reporting, and defect analysis.
- [[autoregressive-image-generation|Autoregressive image generation]] — Autoregressive image generation models produce images by predicting discrete tokens sequentially, similar to how language models predict text tokens. This contrasts with diffusion models, which denoise images iteratively.
- [[math-competitions-llm-evaluation|Math Competitions LLM Evaluation]] — Using mathematics competitions and olympiads as uncontaminated benchmarks for evaluating LLM reasoning capabilities. Key principles: novel problems, multiple runs, item-response theory modeling, cost-performance analysis.
- [[chain-of-thought|Chain of Thought]] — A prompting technique and training methodology where language models generate intermediate reasoning steps before producing a final answer. CoT prompting ("think step by step") significantly improves performance on complex reasoning tasks. Modern reasoning models are explicitly trained to generate these "reasoning traces" as a core behavior.
- [[claude-code|Claude Code]] — Anthropic's agentic coding tool that runs in the terminal, IDE, desktop app, and web. It reads the entire codebase, understands project structure, and autonomously edits files, runs shell commands, and manages Git from natural language instructions. The flagship model is Sonnet 4.6 with Opus 4.6 for complex reasoning.
- [[code-generation-with-llms|Code Generation with LLMs]] — The use of large language models to generate, complete, or modify source code. Code generation tools analyze context (surrounding code, comments, open files) to produce syntactically correct and semantically relevant suggestions. The most common form is inline autocompletion, but the field also includes chat-based generation, test generation, documentation, and bug fixing.
- [[computational-irreducibility|Computational Irreducibility]] — A concept from Stephen Wolfram's work stating that for certain computational processes, there is no way to predict the outcome without essentially running the computation step by step. This implies fundamental limits to what trainable systems like neural networks can learn — they can capture patterns and regularities but cannot shortcut truly irreducible computations.
- [[context-engineering|Context Engineering]] — The discipline of selecting, organizing, and exposing the right subset of system knowledge to an LLM so it can operate effectively. Context Engineering sits between Prompt Engineering (optimizing a single input) and Harness Engineering (the full infrastructure), forming the middle layer: Prompt ⊂ Context ⊂ Harness.
- [[context-rot|Context Rot]] — The degradation of LLM performance as context window utilization increases, even when the theoretical token limit has not been reached. Models begin losing coherence, forgetting earlier constraints, contradicting previous responses, and hallucinating details once context usage crosses approximately 60% of the advertised window. Also known as the "lost in the middle" problem in research literature.
- [[cross-attention|Cross-Attention]] — A variant of attention where the query sequence comes from one input and the key/value sequences come from a different input. Used in encoder-decoder transformers (where decoder queries attend to encoder outputs) and in multimodal models like Stable Diffusion (where image features attend to text prompts).
- [[cuda|CUDA]] — CUDA (Compute Unified Device Architecture) is NVIDIA's parallel computing platform and programming model that enables developers to use NVIDIA GPUs for general-purpose processing. It extends C/C++ with keywords like `__global__` for kernel functions and provides APIs for memory management, thread synchronization, and profiling.
- [[cynefin-framework|Cynefin Framework]] — A sense-making framework that categorizes problems into five domains: Clear (obvious cause-effect), Complicated (expert analysis needed), Complex (understood only in retrospect), Chaotic (no perceivable order), and Confusion (uncertain which domain applies). Chris Richardson applied it to argue that GenAI coding agents operate in the Complex domain, requiring safe experimentation rather than standardized best practices.
- [[data-contamination|Data Contamination]] — The presence of evaluation task data or labeled examples in a model's pretraining corpus. When test sets or benchmark examples appear in training data, models may appear to generalize better than they actually do, leading to inflated performance estimates.
- [[deepseek-sparse-attention|DeepSeek Sparse Attention]] — A learned sparse attention mechanism that uses an indexer and token selector to decide which past tokens to revisit, rather than relying on hard-coded locality (as in Sliding Window Attention) or attending to all tokens. The model learns a sparse pattern that preserves important long-range dependencies while skipping irrelevant tokens.
- [[deep-document-understanding|Deep Document Understanding]] — AI techniques that parse and comprehend complex document formats beyond simple text extraction, including structure, layout, images, tables, and relationships. Used in RAG systems like RAGFlow for processing PDFs and images.
- [[diffusion-models|Diffusion Models]] — Diffusion models are a class of generative models that learn to reverse a gradual noising process. Starting from random noise, they iteratively denoise to generate high-quality samples (images, audio, video). They have become the dominant approach for image generation, supplanting GANs and VAEs.
- [[distillation|Distillation]] — In the context of large language models, distillation refers to training smaller "student" models on outputs (or reasoning traces) generated by larger "teacher" models. Unlike classical knowledge distillation (which uses teacher logits), LLM distillation typically involves supervised fine-tuning on high-quality teacher-generated data.
- [[document-ai|Document AI]] — The broader field encompassing AI techniques for understanding, parsing, and extracting information from documents in various formats. Subset includes deep document understanding focused on complex parsing.
- [[dspy|DSPy]] — A declarative framework for building modular AI software with language models. DSPy (Declarative Self-improving Python) enables developers to write structured, typed code with natural-language signatures rather than brittle prompt strings. The framework compiles high-level programs into optimized prompts or model weight updates via built-in optimizers.
- [[eigenvalue|Eigenvalue]] — A scalar (λ) associated with a square matrix representing how the matrix scales space along certain directions. Found by solving characteristic equation det(A - λI) = 0. Properties reveal matrix behavior: positive (stretches), negative (flips), zero (collapses), large magnitude (strong transformation). Applications: PCA (variance captured by principal components), matrix analysis (stability, invertibility), linear systems (differential equations, dynamical stability), spectral clustering, kernel methods. Foundation for dimensionality reduction and understanding model behavior.

- [[eight-factor-agent|8-Factor Agent Methodology]] — A set of best practices for building production AI applications that incorporate LLM agents. - [[feed-forward-network|Feed-Forward Network]] — A position-wise two-layer MLP within each Transformer block that transforms the contextual representations produced by attention into richer, task-specific features. It processes each token independently — attention mixes information across tokens; the FFN transforms information within each token.
- [[foundation-model|Foundation Model]] — A large-scale AI model trained on broad data at scale such that it can be adapted to a wide range of downstream tasks. Foundation models are typically trained via self-supervised learning on massive unlabeled datasets and then fine-tuned or prompted for specific applications.
- [[generative-ai-platform|Generative AI Platform]] — The production infrastructure and architecture for deploying generative AI applications at scale. Based on [[chip-huyen|Chip Huyen]]'s analysis of how companies actually deploy LLM applications.
- [[grouped-query-attention|Grouped-Query Attention]] — An attention mechanism that reduces the memory footprint of the KV cache by sharing key and value heads across multiple query heads. Instead of every query head having its own K and V projections (as in standard Multi-Head Attention), GQA groups query heads together so that each group shares a single K/V head.
- [[harness-engineering|Harness Engineering]] — The discipline of designing the execution environment, constraints, feedback loops, and orchestration infrastructure that surrounds an AI model (especially an agent) to make it reliable, safe, and productive in real-world engineering. Coined by Mitchell Hashimoto (HashiCorp co-founder) and popularized by OpenAI's 2026 report on Codex harnesses. Core metaphor: a harness turns a powerful but unpredictable "wild horse" (the model) into a reliable "steed" (the production system).
- [[http-cookie-handling|HTTP Cookie Handling]] — The set of behaviors and specifications governing how HTTP cookies are set, parsed, transmitted, and stored across web browsers and server-side programming languages. Due to historical evolution through multiple RFCs (2109, 2965, 6265, 6265bis) and Postel's Law-inspired tolerance, implementations diverge significantly in their acceptance of special characters, control characters, and Unicode in cookie names and values.
- [[hybrid-attention|Hybrid Attention]] — An architectural approach that replaces most full-attention or self-attention layers with cheaper linear-attention or state-space modules (e.g., Gated DeltaNet, Mamba-2), while retaining periodic heavy attention layers for exact retrieval. This trades some modeling quality for dramatically better long-context efficiency.
- [[in-context-learning|In-Context Learning]] — The ability of large language models to learn new tasks from examples provided in the prompt, without any parameter updates. By including a few input-output examples (few-shot) or instructions (zero-shot) in the context, the model adapts its behavior to perform the desired task.
- [[inference-time-scaling|Inference-Time Scaling]] — Increasing computational resources during inference (rather than training) to improve LLM output quality. The analogy is giving a human more time to think through a complex problem. For reasoning models, this means generating more tokens — intermediate reasoning steps — before arriving at a final answer.
- [[kv-caching|KV caching]] — KV caching is an inference optimization technique in transformer models that stores the Key (K) and Value (V) matrices from previous tokens, avoiding redundant recomputation during autoregressive generation. This enables providers to offer cached input tokens at roughly 10x lower cost.
- [[layer-normalization|Layer Normalization]] — A normalization technique that normalizes across the embedding dimensions of a single token (rather than across the batch), making it suitable for variable-length sequences in transformers. It prevents exploding or vanishing values in deep stacks and is a critical component of stable transformer training.
- [[llm-agents|LLM Agents]] — Systems that use LLMs as reasoning engines to autonomously plan, use tools, and execute multi-step tasks. Agents go beyond single-turn prompting by incorporating memory, tool use, and feedback loops.
- [[ml-mathematics|ML Mathematics]] — The mathematical foundations required to understand and implement machine learning algorithms: linear algebra, calculus, probability theory, and optimization. While modern frameworks abstract away much of the math, deep understanding enables debugging, research, and algorithm design.

- [[model-context-protocol|Model Context Protocol]] — An open-source protocol developed by Anthropic that standardizes how AI systems connect to external data sources, tools, and prompts. Uses a client-server architecture where MCP servers expose resources and tools, and MCP clients (e.g., Cursor, Claude Desktop) consume them.
- [[model-scaling|Model Scaling]] — The engineering discipline of training and serving large language models efficiently across multiple accelerators (TPUs, GPUs) while maintaining linear throughput growth. Model scaling addresses how to parallelize computation, minimize communication overhead, and optimize memory usage so that adding more hardware yields proportional performance gains.
- [[model-fine-tuning|Model Fine-Tuning]] — The process of taking a pre-trained model and further training it on domain-specific data to improve performance for particular tasks. Standard practice for adapting general models, with techniques from full fine-tuning to parameter-efficient methods like LoRA.
- [[multi-head-attention|Multi-Head Attention]] — An extension of self-attention that uses multiple independent attention heads operating in parallel. Each head can potentially learn to focus on different parts of the input sequence, capturing various aspects or relationships within the data. The outputs from all heads are concatenated and linearly projected to produce the final result.
- [[multi-head-latent-attention|Multi-Head Latent Attention]] — An attention mechanism pioneered by DeepSeek that compresses the KV cache by storing a low-dimensional latent representation instead of full key and value tensors. During inference, the full K and V matrices are reconstructed on-the-fly from the cached latent vector, dramatically reducing memory usage at scale.
- [[on-policy-distillation|On-policy distillation]] — On-policy distillation is a post-training method that samples trajectories from a student model and grades each token with a teacher model, combining the on-policy relevance of RL with the dense reward signal of supervised distillation.
- [[paged-attention|Paged attention]] — A memory management technique for LLM inference that divides the KV cache into fixed-size blocks (pages), similar to virtual memory paging in operating systems. Introduced in vLLM, it dramatically improves GPU memory utilization and throughput by eliminating memory fragmentation and enabling dynamic block allocation.
- [[paper-club|Paper Club]] — A regular, peer-led reading group where participants pre-read and discuss research papers. The paper club model emphasizes shared accountability, spaced repetition, critical evaluation, and community building. It is a primary learning methodology for practitioners who want to stay current with research without the overhead of formal academic training.
- [[pi-agent|Pi Agent]] — A minimal, opinionated coding agent created by Mario Zechner. Pi's philosophy is radical minimalism: a sub-1000-token system prompt, only 4 core tools (read, write, edit, bash), no MCP support, no built-in plan mode, no background bash, and YOLO-by-default execution. Despite (or because of) its minimalism, Pi outperforms Claude Code and Codex on Terminal-Bench 2.0.
- [[progressive-disclosure|Progressive Disclosure]] — A context management pattern where agents receive only lightweight metadata or indexes initially, and load detailed knowledge on demand when executing specific tasks. This "pull mode" contrasts with "push mode" (pre-loading all relevant context), dramatically reducing context window consumption and preventing context rot.
- [[quantization|Quantization]] — The practice of reducing the numerical precision of a model's weights (and optionally activations) from higher-precision formats (e.g., float32, float16) to lower-precision formats (e.g., int8, int4, float8). This compresses model size, reduces memory bandwidth, and can accelerate inference on supported hardware.
- [[rag|RAG (Retrieval Augmented Generation)]] — Technique allowing LLMs to integrate external data in real-time, improving accuracy for Q&A, summarization, and content generation. Key components include fine-tuning models, optimizing embeddings, and refining query-document relevance.

- [[rag-security|RAG Security]] — The set of security concerns and mitigations specific to Retrieval-Augmented Generation systems. RAG security addresses risks that arise when LLMs are given access to external data sources, including unauthorized data access, sensitive information disclosure, prompt injection via retrieved content, and data poisoning.
- [[ralph-loop|Ralph Loop]] — A technique for running AI coding agents autonomously by repeatedly executing the same prompt in a loop. The agent reads a PRD (Product Requirements Document) and a progress file, implements one task, commits the changes, and updates progress. Named after Ralph Wiggum from The Simpsons by creator Geoffrey Huntley.
- [[reasoning-llms|Reasoning LLMs]] — Large language models that employ extended test-time compute — chain-of-thought reasoning, lookahead search, or reinforcement learning — to solve complex problems. Rather than generating an answer in a single pass, reasoning models "think longer" at inference time, often producing intermediate reasoning steps before a final answer.
- [[retrieval-augmented-generation|Retrieval Augmented Generation (RAG)]] — An architecture pattern for LLM applications where the model retrieves relevant documents or facts from an external knowledge base before generating a response. RAG grounds the LLM in specific, up-to-date, or private information that was not in its training data.
- [[rmsnorm|RMSNorm]] — Root Mean Square Layer Normalization. A normalization technique that scales inputs by their RMS without centering (subtracting the mean), unlike standard LayerNorm. Used as pre-normalization in architectures like Llama.
- [[rotary-positional-embedding|Rotary positional embedding]] — A positional encoding method for transformers that encodes token position by rotating the embedding vector in 2D subspaces, with a different rotation angle at each position. Introduced in the RoPE paper (Su et al., 2021) and adopted in Llama, PaLM, and other modern models.
- [[self-attention|Self-Attention]] — A mechanism that enables a model to weigh the importance of different elements in an input sequence and dynamically adjust their influence on the output. In self-attention, each position in the sequence attends to all other positions in the same sequence, producing a context-aware representation.
- [[self-improving-agent|Self-Improving Agent]] — An AI agent architecture that automatically distills experience from completed tasks into reusable, persistent knowledge (typically Skills or rules), enabling the agent to perform better on future similar tasks without human intervention. The learning loop typically involves a background review process that analyzes session traces, identifies pitfalls, and patches the agent's procedural memory.

- [[sliding-window-attention|Sliding Window Attention]] — An attention mechanism that restricts each token to attend only to a fixed-size local neighborhood of recent tokens, rather than the entire prefix. - [[swiglu|SwiGLU]] — Swish-Gated Linear Unit. An activation function used in modern transformer architectures (notably Llama and PaLM) that combines a Swish-activated gate with a linear projection via element-wise multiplication. Defined as SwiGLU(x) = Swish(xW + b) ⊙ (xV + c).

### Concepts

- [[position-embedding|Position Embedding]] — Technique injecting sequence order into permutation-equivariant transformers. Approaches: absolute embeddings, relative encodings, sinusoidal encodings. Essential for transformers lacking inherent sequential processing.

- [[wordpiece-tokenization|WordPiece Tokenization]] — Subword tokenization algorithm splitting words into meaningful units (e.g., "walk" + "##ing"). Used by BERT. Balances vocabulary size with rare word handling. Preserves morphological patterns.

- [[gradient-clipping|Gradient Clipping]] — Technique preventing exploding gradients by limiting gradient magnitude. Methods: clip by norm, clip by value. Stabilizes training in deep networks and RNNs.

- [[chain-rule|Chain Rule]] — Calculus principle computing derivatives of composite functions. Foundation for backpropagation, decomposes gradient computation through multiple layers.

- [[activation-function|Activation Function]] — Non-linear functions introducing complexity into neural networks. Common: ReLU, sigmoid, tanh, GELU, softmax. Properties: derivative, range, zero-centering affect gradient flow.

### Concepts

- [[inference-time-compute-scaling|Inference-Time Compute Scaling]] — Improving LLM reasoning by increasing compute during inference without modifying weights. Methods: chain-of-thought, majority voting, beam search, process reward models, budget forcing ("Wait" tokens), MCTS. Key findings: small models + scaling can outperform larger models. Trade-offs: accuracy vs latency/cost. No single technique best for all tasks.

- [[reasoning-models|Reasoning Models]] — LLMs designed for multi-step problems through structured thought processes. Approaches: inference-time scaling, pure RL, RL+SFT, SFT+distillation. DeepSeek R1 (Jan 2025) case study. Methods explicitly generate intermediate steps or handle them internally.

- [[small-language-models|Small Language Models]] — Compact models (<10B parameters) with LLM capabilities. Characteristics: smaller training data, lower compute, faster latency, edge deployment. Microsoft Phi-3 innovation: high-quality curated data beats massive raw data. Use cases: simple tasks, offline, cost-sensitive. Limitations: not for complex reasoning.

- [[synthetic-data|Synthetic Data]] — Training data generated by AI models (LLMs) rather than real-world sources. Microsoft's TinyStories: millions of children's stories from 3,000 words. CodeTextbook: filtered public data with educational value. Benefits: high quality, tailored content. Trade-off: may lack diversity vs raw web data.

- [[tpu|TPU]] — Google's ASIC for ML matrix operations. 80-95% of transformer compute is matrix multiplication. Architecture: systolic array (256x256 in TPUv1, 2x2 educational), weight-stationary design. Faster than GPUs for pure ML due to specialization. Cannot render graphics.

- [[systolic-array|Systolic Array]] — Grid of Processing Elements for matrix multiplication. Each PE: multiply-accumulate in one cycle. Weights stationary, inputs flow right, sums flow down. Name from systolic blood pressure analogy (coordinated beats). TPUv1: 256x256 array.

- [[distributed-training|Distributed Training]] — Training large models across thousands of GPUs. Methods: data parallelism (split batches), model parallelism (split model). Llama 3.1 405B: tensor=8, pipeline=16, data=8-128. Considerations: network topology (NVLink intra-node, InfiniBand cross-node). Tricks: gradient accumulation, activation checkpointing.

- [[data-parallelism|Data Parallelism]] — Splitting batches across GPUs, each holds full model. All-reduce sums gradients, keeps models synchronized. Ring all-reduce maximizes bandwidth. Scales batch size, limited by model memory. Best for slower cross-node communication.

- [[model-parallelism|Model Parallelism]] — Splitting model across GPUs when too large for single GPU. Two types: pipeline (layer-wise), tensor (operation-level). Pipeline: sequential stages, minimize communication. Tensor: split matrix ops, requires fast interconnects (NVLink).

- [[pipeline-parallelism|Pipeline Parallelism]] — Distributing layers across GPUs (e.g., layers 1-4 on GPU0, 5-8 on GPU1). Challenge: pipeline bubble (idle time). Solution: micro-batches for parallelization. GPipe: accumulate gradients. PipeDream: update each micro-batch (near-perfect utilization, more inconsistency).

- [[tensor-parallelism|Tensor Parallelism]] — Splitting layer operations across GPUs. Megatron-LM optimization: column→activation→row partitioning reduces synchronization 50%. Requires high-speed interconnects (NVLink). Frequent communication: all-gather, all-reduce.
